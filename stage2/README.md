# 第 2 阶段：先做出第一个全栈小项目

- Day 范围：15～28
- 阶段目标：尽快把你的前端优势接进来，做一个有界面的 AI 应用。

## 阶段日程
- [Day 15](day15.md)：确定项目题目。
  任务：从 AI 文案助手、AI 总结助手、AI 翻译润色助手中选 1 个；写需求清单。
  产出：项目说明文档。
- [Day 16](day16.md)：设计前后端结构：前端 React/Next.js，后端 FastAPI。
  任务：画简单架构图。
  产出：项目目录设计。
- [Day 17](day17.md)：搭建前端页面：输入框、按钮、结果区。
  任务：先用 mock 数据打通界面。
  产出：前端初版页面。
- [Day 18](day18.md)：后端接入本地 Qwen 接口。
  任务：做一个 /generate 接口。
  产出：后端初版。
- [Day 19](day19.md)：前后端联调。
  任务：页面输入内容后能拿到模型输出。
  产出：第一个 AI Web 应用可用。
- [Day 20](day20.md)：优化输出体验：loading、错误提示、复制按钮。
  任务：做基础 UX 优化。
  产出：更完整的界面。
- [Day 21](day21.md)：复盘。
  任务：写项目 README，截图保存。
  产出：项目 v1。
- [Day 22](day22.md)：学习 token、上下文窗口。
  任务：解释为什么长文本不能无脑全塞给模型。
  产出：术语笔记。
- [Day 23](day23.md)：学习 Transformer 的直觉版理解，不追数学。
  任务：画出“输入 → 编码 → 注意力 → 输出”的简图。
  产出：一张图。
- [Day 24](day24.md)：学习 few-shot、zero-shot。
  任务：对比同一任务下两种 prompt 的效果。
  产出：实验记录。
- [Day 25](day25.md)：学习 prompt 防歧义、格式约束。
  任务：改造你的项目 prompt，让输出更稳定。
  产出：项目 v2 prompt。
- [Day 26](day26.md)：学习错误处理、重试、日志。
  任务：给你的后端加异常捕获和日志。
  产出：更稳的接口。
- [Day 27](day27.md)：学习简单成本意识。
  任务：记录一次请求的输入长度、输出长度。
  产出：成本意识表。
- [Day 28](day28.md)：阶段复盘。
  任务：总结“我已经真正掌握了哪些 LLM 基础”。
  产出：阶段总结 1。
