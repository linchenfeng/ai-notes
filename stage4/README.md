# 第 4 阶段：评测与优化

- Day 范围：50～63
- 阶段目标：从“能跑”升级到“更准、更稳”。

## 阶段日程
- [Day 50](day50.md)：学习什么叫评测：正确性、相关性、幻觉、引用准确。
  任务：写一份你的项目评测标准。
  产出：eval_criteria.md。
- [Day 51](day51.md)：构建测试集。
  任务：准备 20～30 个标准问题。
  产出：eval_dataset.json。
- [Day 52](day52.md)：写批量测试脚本。
  任务：让程序自动跑一批问题。
  产出：run_eval.py。
- [Day 53](day53.md)：记录结果。
  任务：保存问题、检索片段、回答、耗时。
  产出：日志文件。
- [Day 54](day54.md)：做第一次评测。
  任务：统计正确率和失败样例。
  产出：评测报告 v1。
- [Day 55](day55.md)：根据评测优化。
  任务：修改 prompt 或检索策略。
  产出：RAG v2。
- [Day 56](day56.md)：重新跑评测。
  任务：对比优化前后差异。
  产出：评测报告 v2。
- [Day 57](day57.md)：学习向量相似度、top-k 的意义。
  任务：用自己的语言讲清楚检索是怎么工作的。
  产出：概念笔记。
- [Day 58](day58.md)：学习关键词检索 vs 向量检索。
  任务：用同一问题做对比实验。
  产出：对比结果表。
- [Day 59](day59.md)：学习混合检索的思路。
  任务：设计一个“关键词 + 向量”的方案。
  产出：方案草稿。
- [Day 60](day60.md)：学习 query rewrite。
  任务：把用户问题改写成更适合检索的形式。
  产出：rewrite demo。
- [Day 61](day61.md)：学习 rerank 是干什么的。
  任务：做一个最简单的二次排序实验。
  产出：实验记录。
- [Day 62](day62.md)：回头看 Embedding 理论。
  任务：补一篇“为什么向量可以表达语义接近”。
  产出：原理理解文档。
- [Day 63](day63.md)：阶段复盘。
  任务：给 RAG 项目做一次整体整理。
  产出：RAG 项目可投简历版。
