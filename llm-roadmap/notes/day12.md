## 我现在理解了什么
- 配置和业务逻辑应该分开
- API Key 不应该写死在代码里
- .env 很适合本地开发
- .env.example 用来提供配置模板
- config.py 是统一读取配置的入口
- 真实项目接外部 API 前，应该先检查配置是否齐全
