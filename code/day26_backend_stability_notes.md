# Day 26：后端稳定性改造记录

## 今日目标
- 给后端加异常捕获和日志
- 让接口在网络抖动时更稳（重试）

## 已完成改造
- 在 `backend/day18_main.py` 增加统一日志配置（支持 `LOG_LEVEL`）。
- 在 `/generate` 增加请求级日志：开始、失败、成功、耗时（ms）与 `request_id`。
- 增加全局异常处理：
  - `RequestValidationError` 返回统一 JSON 错误结构（422）
  - `Exception` 返回统一 JSON 错误结构（500）
- 给远程模型调用增加重试机制：
  - 重试场景：`429/5xx`、超时、网络连接失败
  - 重试次数：最多 2 次
  - 退避策略：`1s -> 2s`（上限 4s）

## 最小验证建议
1. 正常请求 `/generate`，确认能返回 `ok=true`。
2. 故意传非法参数（如缺少 `text`），确认返回 `ok=false` 且状态码 422。
3. 临时关闭网络或改错 API 地址，确认接口能重试并最终返回可读错误信息。
4. 观察终端日志是否包含 `request_id` 与耗时。

## 一句话复盘
今天把“异常捕获 + 日志 + 重试”落到了接口层，让这个总结应用在真实联调里更抗抖动、更好排障。
