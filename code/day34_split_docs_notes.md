# Day 34：切分脚本记录

## 今日目标

- 把 Day32 准备的资料库真正切成多个 chunk
- 为后续 embedding、索引和检索准备统一输入

## 今日产出

- 切分脚本：[day34_split_docs.py](/Volumes/学习资料/ai-notes/code/day34_split_docs.py)

## 当前实现了什么

- 复用 Day33 的文档读取逻辑，不重复写读取代码
- 支持从 `code/docs` 目录加载 `.md/.txt` 文档
- 按固定字符窗口切分文档
- 支持 overlap，减少切分边界断裂问题
- 输出每个 chunk 的来源信息、字符区间和内容预览

## 实际验证结果

- 成功加载 21 篇文档
- 成功切分得到 24 个 chunk

## 这一步的意义

到 Day34 为止，RAG 上游链路已经从：

`文档目录 -> 读取文档`

推进到了：

`文档目录 -> 读取文档 -> 切成 chunk`

这意味着后面 Day37 做 embedding 时，不再直接面对整篇文档，而是面对已经切好的知识单元。

## 当前版本的边界

- 现在先用的是“固定长度 + overlap”切分
- 还没有实现按段落切或按标题切
- 还没有把 chunk 结果写入文件，只是先在命令行输出

## 下一步衔接

- Day35 可以回头检查 chunk 是否合理
- 如果发现切分质量不够，再继续优化切分策略
