# Day 27：成本意识表

## 今日目标
- 记录一次请求的输入长度、输出长度
- 在项目里建立最基础的成本意识

## 已完成改造
- 后端 `POST /generate` 响应新增：
  - `input_length`
  - `output_length`
- 后端成功日志新增：
  - 输入长度
  - 输出长度
- 前端页面新增“成本意识表（最近一次）”，可以直接看到：
  - 请求模式
  - 总结风格
  - 输入长度
  - 输出长度
  - 本次耗时

## 今天学到的点
- 即使还没做精确 token 统计，先记录输入和输出的字符长度，也能建立最基础的“请求成本”感觉。
- 当输入变长、输出要求变细时，模型调用成本和响应时间通常都会一起上升。
- 这一步以后会直接用在更复杂的项目里，比如长文本总结、RAG 检索问答和多轮对话。

## 一句话复盘
今天把“记录输入长度和输出长度”落到了真实页面和接口里，开始对模型调用的成本有最基础的量化认知。
