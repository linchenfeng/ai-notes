# Day 25 项目 v2 Prompt 改造记录

## 今日目标
改造 AI 总结助手的 prompt，降低歧义并增强格式稳定性。

## 改造文件
- `code/day16_20_ai_summary_app/backend/day18_main.py`

## v1 问题（改造前）
- 模型偶尔复述规则文字，而不是直接给结果。
- `bullets` 有时输出编号、段落或超过 5 条。
- `structured` 有时缺少固定键名，或夹杂额外说明。

## v2 改造策略
1. 强化系统提示词（System Prompt）
- 明确“只能基于原文事实，不要编造”。
- 明确“必须遵守格式，不要输出额外文字和代码块”。

2. 分风格细化用户提示词
- `brief`：限制单段、字数范围、禁用引导语。
- `bullets`：限制 3~5 条、每条必须 `- ` 开头、禁用编号和标题。
- `structured`：强制三行键名（背景/关键动作/结果）并禁止额外输出。

3. 增加轻量输出规范化
- `brief`：合并换行，保持单段。
- `bullets`：统一转换为 `- ` 列表并截断到最多 5 条。
- `structured`：缺失键名时补齐占位文案，保证三行完整。

## v2 价值
- 输出更稳定，前端展示逻辑更简单。
- 风格切换后，格式一致性更高。
- 减少“模型发挥过度”带来的异常结果。
