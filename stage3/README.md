# 第 3 阶段：进入最重要的能力——RAG

- Day 范围：29～49
- 阶段目标：学会做知识库问答，这是初学者最值得投入的主线。

## 阶段日程
- [Day 29](day29.md)：学习 RAG 总流程：读文档、切分、向量化、检索、生成。
  任务：画 RAG 流程图。
  产出：day29_rag_flow.md。
- [Day 30](day30.md)：学习什么是 Embedding。
  任务：用自己的话写清楚“生成模型”和“向量表示”的区别。
  产出：解释文档。
- [Day 31](day31.md)：学习 chunking。
  任务：拿一篇长文，按 3 种方式切分。
  产出：切分实验。
- [Day 32](day32.md)：准备文档集。
  任务：选一个主题，准备 20～30 篇资料。推荐：前端、数据库、AI 学习资料都可以。
  产出：docs/ 资料库。
- [Day 33](day33.md)：写文档读取脚本。
  任务：先支持 txt/md。
  产出：day33_load_docs.py。
- [Day 34](day34.md)：写切分脚本。
  任务：把文档切成多个 chunk。
  产出：split_docs.py。
- [Day 35](day35.md)：复盘。
  任务：检查切分结果是否合理。
  产出：切分样本。
- [Day 36](day36.md)：学习向量数据库的基本概念。
  任务：选一个轻量方案先上手。
  产出：本地索引环境。
- [Day 37](day37.md)：实现 Embedding。
  任务：把文档 chunks 转成向量。
  产出：embed_docs.py。
- [Day 38](day38.md)：实现向量入库。
  任务：写索引构建脚本。
  产出：build_index.py。
- [Day 39](day39.md)：实现相似度检索。
  任务：输入问题，返回 top-k chunks。
  产出：retriever.py。
- [Day 40](day40.md)：把检索结果塞给模型生成回答。
  任务：完成最小 RAG 闭环。
  产出：命令行版知识库问答。
- [Day 41](day41.md)：给答案加“引用来源”。
  任务：输出文档标题或 chunk 来源。
  产出：带引用的回答。
- [Day 42](day42.md)：做简单前端界面。
  任务：把 RAG 接到网页上。
  产出：RAG Web Demo。
- [Day 43](day43.md)：学习“有证据再回答，没证据就拒答”。
  任务：加一个拒答机制。
  产出：更稳的 RAG v2。
- [Day 44](day44.md)：学习 metadata。
  任务：给文档加标题、标签、来源。
  产出：结构化索引数据。
- [Day 45](day45.md)：优化 retrieval 的 prompt。
  任务：让模型更依赖检索内容回答。
  产出：更稳的生成模板。
- [Day 46](day46.md)：做 15～20 个测试问题。
  任务：人工测试正确率。
  产出：测试题集。
- [Day 47](day47.md)：记录 bad case。
  任务：整理至少 8 个错误示例。
  产出：rag_badcases.md。
- [Day 48](day48.md)：调参数：chunk size、top-k。
  任务：比较不同参数的效果。
  产出：实验记录表。
- [Day 49](day49.md)：阶段复盘。
  任务：整理出你第一个完整 RAG 项目仓库。
  产出：项目 v1 可展示版。
