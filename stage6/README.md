# 第 6 阶段：Agent、流程编排、部署

- Day 范围：78～87
- 阶段目标：在已有能力上做复杂一点，但不追求“很炫”，追求“稳定”。

## 阶段日程
- [Day 78](day78.md)：理解 Agent 和普通工具调用的区别。
  任务：写出“单步工具”与“多步任务”的差异。
  产出：概念对比表。
- [Day 79](day79.md)：实现一个简单多步流程。
  任务：先分析问题 → 再调工具 → 再总结结果。
  产出：简易 Agent v1。
- [Day 80](day80.md)：给 Agent 加步骤日志。
  任务：显示每一步做了什么。
  产出：可观察版 Agent。
- [Day 81](day81.md)：学习超时、重试、fallback。
  任务：让 Agent 失败时不直接崩。
  产出：健壮版 Agent。
- [Day 82](day82.md)：看一个 Agent 框架，但只做轻量了解。
  任务：对比你手写版和框架版的区别。
  产出：框架对比笔记。
- [Day 83](day83.md)：了解 MCP 概念。
  任务：知道它是干什么的即可，不要求深入。
  产出：MCP 入门笔记。
- [Day 84](day84.md)：部署一个项目上线。
  任务：把你的 AI 应用或 RAG 项目部署到线上。
  产出：在线地址。
- [Day 85](day85.md)：加环境变量、日志、错误上报。
  任务：补齐工程化细节。
  产出：可上线项目 v2。
- [Day 86](day86.md)：做一次简单压力测试。
  任务：模拟多次请求，记录耗时。
  产出：性能记录表。
- [Day 87](day87.md)：阶段复盘。
  任务：确定最终要保留的 2～3 个项目。
  产出：作品集清单。
